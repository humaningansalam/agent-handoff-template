# Product (Example)

Use this file for product intent and scope.
Do not put detailed API, schema, or runtime state logic here.

## One-Sentence Summary

Describe the product/project in one clear sentence.

## Problem

Describe the problem being solved.

Prompt:
- What is broken, missing, or too expensive today?
- Why does this project exist now?

## Users / Stakeholders

List the primary users or stakeholders.

Suggested format:
- **Primary user**: who they are and what they need
- **Secondary user**: who else matters and why

## Current Phase

State the current phase and what it is trying to achieve.

Suggested fields:
- Current phase:
- Phase goal:
- What is intentionally deferred:

## In Scope

List what this phase/version should include.

## Out of Scope

List what should not be implemented or assumed in this phase.

## Success Signals

List a few concrete outcomes that would indicate success.

Examples:
- activation target
- latency or reliability target
- user adoption signal
- operator efficiency gain
