# Product Requirements Document (Index Example)

This project keeps PRD context split into small role-based documents.

Do not read every PRD file by default.
Read only the files listed under `## Context Docs` in the active task.

## Read Map

- Product purpose, users, phase, scope
  - `docs/prd/product.md`
- Rules, guardrails, constraints, business policies
  - `docs/prd/policies.md`
- System structure, boundaries, deployment overview
  - `docs/prd/architecture.md`
- State transitions, runtime flows, failure handling
  - `docs/prd/runtime.md`
- Data/API/event/file contracts
  - `docs/prd/contracts.md`

## Reading Rule

Start with this index only when broader product/project context is needed.
Then read only the specific files required by the task.
